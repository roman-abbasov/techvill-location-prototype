# Обновление текста «Важно о прототипе»

## Цель

Сделать пояснение на странице «Прототип» более информативным: добавить инструкцию по просмотру рекомендаций и предупредить о возможных расхождениях демонстрационных данных с действительностью.

## Представление

Существующий блок «Важно о прототипе» сохраняет оформление, заголовок, информационную иконку и расположение. Текст внутри блока разбивается на три отдельных абзаца:

1. «Здесь я подтянул Яндекс API со своим ключом и хотел наиболее наглядно продемонстрировать прототип. Помимо прочего пытался на основе **Yandex Maps API Heatmap Module** отрисовать именно тепловую карту. Далеко от идеала (я не программист), но я уверен с командой на обсуждении можно будет прийти либо к такому, но более приемлемому варианту либо вообще пересмотреть формат отображения и отрисовки.»
2. «Для просмотра как работают рекомендации **нужно кликнуть** в предложенную локацию.»
3. «**Сразу оговорюсь** - какие-то данные могут не совпадать с действительностью (например, я не вписывал в прототип точный список розничных магазинов и алгоритм может врать, что рядом другой магазин). Цель была и есть - просто наглядно продемонстрировать моё видение функционала.»

Точки в конце предложений и дефисы сохраняются в пользовательской формулировке. Новые компоненты и интерактивное поведение не добавляются.

## Проверка

Тест страницы прототипа должен подтвердить:

- наличие всех трёх абзацев;
- полужирное отображение фраз «Yandex Maps API Heatmap Module», «нужно кликнуть» и «Сразу оговорюсь»;
- сохранение заголовка «Важно о прототипе»;
- отсутствие подписи «AI location intelligence».

После изменения выполняются целевой тест, полный набор тестов и production-сборка.
